# <a name="error-unable-to-start-debugging-on-the-web-server"></a>Ошибка. Не удается запустить отладку на веб-сервере
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Последнюю версию этого раздела можно найти в [ошибка: не удалось начать отладку на веб-сервере](https://docs.microsoft.com/visualstudio/debugger/error-unable-to-start-debugging-on-the-web-server).  
  
При попытке отладить приложение ASP.NET, запущенное на веб-сервере, может появиться следующее сообщение об ошибке: "Не удается запустить отладку на веб-сервере".
  
Во многих случаях эта ошибка возникает, поскольку IIS не настроен правильно.

##  <a name="vxtbshttpservererrorsthingstocheck"></a> Проверьте конфигурацию IIS

После выполнения действия, чтобы устранить неполадки, описанные здесь и перед повторной попыткой для отладки, может потребоваться сброс IIS. Это можно сделать, открыв командную строку администратора и введя `iisreset`, или это можно сделать в диспетчере служб IIS. 

* Остановка и перезапуск пулов приложений, а затем повторите попытку.

    Пул приложений была остановлена или другой внесенные изменения конфигурации может потребоваться остановить и перезапустить пул приложений.
    
    > [!NOTE]
    > Если отслеживает остановки пула приложений, может потребоваться удалить модуль переопределения URL-адрес с помощью панели управления и повторно установить его с помощью установщика платформы Web (Майкрософт). Это может вызвать проблемы после обновления значительного объема системной.

* Проверьте конфигурацию пула приложений, его при необходимости исправьте и повторите попытку.

    Если пароль учетные данные изменились, может потребоваться обновить их в пуле приложений. Кроме того Если вы недавно установили ASP.NET, неправильная версия ASP.NET можно настроить пул приложений. Устраните проблему и перезапустите пул приложений.
    
* Убедитесь, что папку веб-приложение имеет необходимые разрешения.

    Убедитесь, что вы предоставляете IIS_IUSRS или учетная запись IUSR (или конкретного пользователя, связанного с пулом приложений) на чтение и выполнение права для папки веб-приложения. Устраните проблему и перезапустите пул приложений.

* Если вы используете файл HOSTS с локальных адресов, попробуйте использовать петлевой адрес вместо IP-адреса компьютера.

* Откройте страницу localhost в браузере.

     Если службы IIS не установлены правильно, должны отобразиться ошибки при вводе `http://localhost` в браузере.
     
     Сведения о развертывании в службах IIS, см. в разделе [Remote Debugging ASP.NET на удаленном компьютере IIS](../debugger/remote-debugging-aspnet-on-a-remote-iis-7-5-computer.md) или для ASP.NET Core, [публикация в службах IIS](https://docs.asp.net/en/latest/publishing/iis.html)).

* Убедитесь, что необходимая версия ASP.NET установлен на сервере IIS.  См. в разделе [развернуть приложение ASP.NET](../debugger/remote-debugging-aspnet-on-a-remote-iis-7-5-computer.md#BKMK_deploy_asp_net) или для ASP.NET Core, [публикация в службах IIS](https://docs.asp.net/en/latest/publishing/iis.html)).

* Создайте простое приложение ASP.NET на сервере.

     Если не удается получить приложения для работы с отладчиком, попробуйте создать простое приложение ASP.NET локально на сервере и попробуйте отладить базовое приложение. Если базовое приложение можно отлаживать, который может помочь выявить отличия между двумя конфигурациями.
  
* Устраните ошибки проверки подлинности, если используется только IP-адрес

     По умолчанию предполагается, что IP-адреса являются частью Интернет-зоны, и проверка подлинности NTLM для них не выполняется. Если веб-сайт настроен в IIS, чтобы требовать проверку подлинности, проверка подлинности завершится ошибкой. Чтобы устранить эту проблему, можно указать имя удаленного компьютера, а не IP-адрес.
     
## <a name="other-causes"></a>Другие причины

При использовании более ранней версии Visual Studio:

- Перезапустите Visual Studio с повышенными привилегиями и повторите попытку.

    Ошибка в более старых версий (исправлены позже) требуется привилегий в некоторых ASP.NET, в сценариях отладки.
    
- Если запущены несколько экземпляров Visual Studio, повторно откройте проект в одном экземпляре Visual Studio и повторите попытку.
