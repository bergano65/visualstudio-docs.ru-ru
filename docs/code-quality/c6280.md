# <a name="c6280"></a>C6280
Предупреждение C6280: \<переменной > выделяется с помощью \<функция >, а удаляется при помощи \<функция >

 Это предупреждение указывает, что вызывающая функция несогласованно выделяет память с помощью функции из одного семейства выделения памяти и освобождения его с помощью функции из другого семейства выделения памяти. Анализатор проверяет это условие только если `_Analysis_mode_(_Analysis_local_leak_checks_)` указано заметки SAL. По умолчанию данная заметка указывается для кода режима (драйвера) ядра Windows. Дополнительные сведения о заметках SAL см. в разделе [использование аннотаций SAL для сокращения количества дефектов в коде C/C++](../code-quality/using-sal-annotations-to-reduce-c-cpp-code-defects.md).

 Например, это предупреждение будет создано, если память выделяется с помощью `malloc` но освобожденные с помощью `GlobalFree` или `delete`. В определенных случаях несоответствия между массивом `new[]` и скалярные `delete`, вместо этого выводятся более конкретные предупреждения.

## <a name="example"></a>Пример
 Это предупреждение возникает в следующем примере кода.

```cpp
// cl.exe /analyze /c /EHsc /nologo /W4
#include <sal.h>
#include <stdlib.h>

_Analysis_mode_(_Analysis_local_leak_checks_)

void f(int arraySize)
{
    int *pInt = (int *)calloc(arraySize, sizeof (int));
    // code ...
    delete pInt;
}

```

 Чтобы устранить это предупреждение, используйте следующий код:

```cpp
// cl.exe /analyze /c /EHsc /nologo /W4
#include <sal.h>
#include <stdlib.h>

_Analysis_mode_(_Analysis_local_leak_checks_)

void f(int arraySize)
{
    int *pInt = (int *)calloc(arraySize, sizeof (int));
    // code ...
    free(pInt);
}

```

 Разные определения интерфейсов API могут использовать разные кучи. Например `GlobalAlloc` использует кучу системы и `free` использует кучу процесса. Это может вызвать повреждение памяти и аварийно завершает работу.

 Они применяются к `new` / `delete` и `malloc` / `free` механизмов выделения памяти. Чтобы избежать возникновения таких потенциальных несоответствий, используйте методы, предоставляемые по C++ стандартной библиотеки шаблонов (STL). К ним относятся [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class), и [вектор](/cpp/standard-library/vector). Дополнительные сведения см. в разделе [интеллектуальные указатели](/cpp/cpp/smart-pointers-modern-cpp) и [стандартной библиотеки C++](/cpp/standard-library/cpp-standard-library-reference).

```cpp
// cl.exe /analyze /c /EHsc /nologo /W4
#include <sal.h>
#include <vector>
#include <memory>

using namespace std;

_Analysis_mode_(_Analysis_local_leak_checks_)

void f(int arraySize)
{
    // use unique_ptr instead of calloc/malloc/new
    unique_ptr<int[]> pInt(new int[arraySize]);

    // code ...

    // No need for free because unique_ptr 
    // cleans up when out of scope.
}

```
