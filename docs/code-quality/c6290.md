# <a name="c6290"></a>C6290
Предупреждение C6290: Битовая операция с логическим результатом:! имеет более высокий приоритет, чем &. Использовать & & или (! () x & y)) вместо

Это предупреждение указывает на возможную неоднозначность в использовании оператора или операторов.

`!` Оператор возвращает логическое значение, и он имеет более высокий приоритет, чем `&`. Побитовое- и (&) оператор принимает два арифметических аргумента. Таким образом одна из следующих ошибок обнаружено:

- Выражение является неправильное использование скобок:

   Так как результат `!` является логическое значение (ноль или один), попытка проверки совпадения двух переменных битов в распространенных будет только окажутся тестирования, что в правой части присутствует младшего бита: `((!8) & 1) == 0`.

- `!` Оператор является неверной и должен быть `~` вместо:

   `!` Оператор имеет логическое значение, хотя `~` оператор имеет возвращает арифметический результат. Эти операторы не являются взаимозаменяемыми, даже если она работает на логическое значение (ноль или один): `((!0x01) & 0x10) == 0x0`, хотя `((~0x01) & 0x10) == 0x10`.

- Бинарный оператор `&` неверна и должен быть `&&`:

   Хотя `&` иногда может быть произвольным с `&&`, они не являются эквивалентными, так как выполняется принудительное вычисление правой части выражения. Могут возникать определенные побочные эффекты в этом типе выражения.

Это можно определить серьезность проблемы без проверки кода. Чтобы убедиться, что происходит проверки следует исследовать код.

## <a name="example"></a>Пример

В следующем коде создается данное предупреждение:

```cpp
void f(int x, int y)
{
  if (!x & y)
  {
    // code ..
  }
}
```

Чтобы устранить это предупреждение, используйте следующий образец кода:

```cpp
void f(int x, int y)
{
  /* When testing that x has no bits in common with y. */
  if (!(x & y))
  {
    // code
  }

  /* When testing for the complement of x in y. */
  if ((~x) & y)
  {
    // code ...
  }
}
#include <windows.h>
void fC(int x, BOOL y )
 {
  /* When y is a Boolean or Boolean result. */
  if ((!x) && y)
  {
    // code ...
  }
}
```

Предупреждение C6317 Если `!` оператор находится в правой части `&` оператор.
