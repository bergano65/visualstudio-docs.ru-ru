# <a name="ca0060"></a>CA0060

Не удалось загрузить сборки с непрямыми.

Это предупреждение возникает, если средство анализа не удается загрузить сборки, на которые косвенно ссылается анализируемая сборка. «Косвенно ссылается» ссылается на базовую сборку, которую одной из ссылок на сборки анализа сборки ссылается на. Пример Если анализ кода (FxCop) анализирует сборку A, и сборка «A» ссылается на сборку «B» сборка «B» ссылается на сборку «C», но «A» не не ссылочную сборку «C», то сборка «C» представляет собой непрямую ссылку и сборка «B» является прямая ссылка .

Невозможность загрузки сборок может вызвать ошибку CA0001 в других местах, так как может привести к непредвиденное состояние. Если вы используете средство анализа управляемого кода из среды Visual Studio, ниже приведены некоторые из причин, которые могут вызвать предупреждение CA0060:

- Входная сборка была переопределена и теперь указывает на сборку, которая ссылается на другие сборки не входят в список ссылок в Visual Studio файл проекта.

- Если проект изменен для вызова пользовательского шага построения, прежде чем FxCopCmd.exe запускается и пользовательского процесса сборки добавляет дополнительные ссылки на входной сборки и создается предупреждение CA0060.

- В C++ можно ссылаться на сборки с помощью ForcedUsing в VCPROJ, ПУТЕМ переопределения командной строки для передачи /FU или добавление #using \<Some.dll > только в исходном.

- Сборка независимых производителей, которая есть закрытые ссылки на другие сборки, чтобы скомпилировать и выполнить код не требуется.

Во всех случаях для устранения этой ошибки является добавление отсутствует ссылка к самому проекту Visual Studio.
