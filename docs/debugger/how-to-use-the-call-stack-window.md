# <a name="view-the-call-stack-and-use-the-call-stack-window-in-the-debugger"></a>Просмотр стека вызовов и использование окна "Стек вызовов" в отладчике

С помощью окна **Стек вызовов** можно просматривать вызовы функций и процедур, которые в данный момент находятся в стеке. В окне **Стек вызовов** показан порядок вызова методов и функций. Стек вызовов хорошо подходит для изучения и анализа потока выполнения приложения.

Когда [отладочные символы](#bkmk_symbols) недоступны для части стека вызовов, **стек вызовов** окно возможно не удастся отобразить правильные сведения об этой части стека вызовов, вместо отображения:

`[Frames below may be incorrect and/or missing, no symbols loaded for name.dll]`

> [!NOTE]
> Окно **Стек вызовов** аналогично перспективе "Отладка" в некоторых интегрированных средах разработки, например Eclipse.

> [!NOTE]
> Отображаемые диалоговые окна и команды меню могут отличаться от описанных здесь в зависимости от текущих параметров или выпуска. Чтобы изменить параметры, выберите в меню **Сервис** пункт **Импорт и экспорт параметров**.  См. в разделе [сбросить параметры](../ide/environment-settings.md#reset-settings).

## <a name="view-the-call-stack-while-in-the-debugger"></a>Просмотр стека вызовов в отладчике

- Во время отладки, в **Отладка** меню, выберите **Windows > стек вызовов**.

  ![Окно стека вызовов](../debugger/media/dbg_basics_callstack_window.png "CallStackWindow")

Желтая стрелка указывает на кадр стека, в котором находится указатель выполнения. По умолчанию сведения о кадре стека отображается в источнике, **"Локальные"**, **"Видимые"**, **Watch**, и **Дизассемблированный код** windows. Изменение контекста отладчика на другой кадр стека, [переключиться на другой кадр стека](#bkmk_switch).

## <a name="display-non-user-code-in-the-call-stack-window"></a>Отображение другого кода в окне стека вызовов

-   Щелкните правой кнопкой мыши в окне **Стек вызовов** и выберите в контекстном меню пункт **Показать внешний код**.

Непользовательский код — это любой код, который не отображается, когда [Just My Code](../debugger/just-my-code.md) включен. В управляемом коде не написанный пользователем код кадров скрыты по умолчанию. Следующая запись отображается вместо кадры не написанный пользователем код:

`[<External Code>]`

## <a name="bkmk_switch"></a> Переключиться на другой кадр стека (изменение контекста отладчика)

1.  В **стек вызовов** окно, щелкните правой кнопкой мыши кадр стека, код и данные, которые вы хотите просмотреть.

    Или можно дважды щелкнуть кадр в **стек вызовов** окно, чтобы переключиться в кадр.

2.  Выберите пункт **Перейти к кадру**.

     Зеленая стрелка с фигурным концом появится рядом с выбранным фреймом стека. Указатель выполнения остается в исходном кадре, который по-прежнему отмечен желтой стрелкой. При выборе команд **Шаг** или **Продолжить** в меню **Отладка** выполнение продолжится с исходного, а не с выбранного кадра.

## <a name="view-the-source-code-for-a-function-on-the-call-stack"></a>Просмотреть исходный код для функции в стеке вызовов

-   В окне **Стек вызовов** щелкните правой кнопкой мыши функцию, исходный код которой нужно увидеть, и выберите пункт **К исходному коду**.

## <a name="run-to-a-specific-function-from-the-call-stack-window"></a>Выполнение до определенной функции из окна «Стек вызовов»

-  В **стек вызовов** окно, выберите функцию, щелкните правой кнопкой мыши, а затем выберите **выполнить до текущей позиции**.

## <a name="set-a-breakpoint-on-the-exit-point-of-a-function-call"></a>Установите точку останова в точке выхода вызова функции

-   См. в разделе [установите точку останова в функции в стеке вызова](../debugger/using-breakpoints.md#BKMK_Set_a_breakpoint_in_the_call_stack_window).

## <a name="display-calls-to-or-from-another-thread"></a>Отображаются вызовы, или из другого потока

-   Щелкните правой кнопкой мыши в окне **Стек вызовов** и выберите пункт **Включить вызовы между потоками**.

## <a name="visually-trace-the-call-stack"></a>Визуальном отслеживании стека вызовов

В Visual Studio Enterprise (только) можно просмотреть карты кода для стека вызовов при отладке.

- В окне **Стек вызовов** откройте контекстное меню. Выберите **Показать стек вызовов на карте кода** (**Ctrl** + **Shift** + **`**).

    Дополнительные сведения см. в разделе [сопоставление методов в стеке вызовов при отладке](../debugger/map-methods-on-the-call-stack-while-debugging-in-visual-studio.md).

![Показать стек вызовов на карте кода](../debugger/media/dbg_basics_show_call_stack_on_code_map.gif "ShowCallStackOnCodeMap")

## <a name="view-the-disassembly-code-for-a-function-on-the-call-stack-c-c-visual-basic-f"></a>Просмотреть дизассемблированный код для функции в стеке вызовов (C#, C++, Visual Basic, F#)

-   В окне **Стек вызовов** щелкните правой кнопкой мыши функцию, дизассемблированный код которой нужно увидеть, и выберите пункт **К дизассемблированному коду**.

## <a name="change-the-optional-information-displayed"></a>Изменить отображение дополнительных сведений

-   Щелкните правой кнопкой мыши в **стек вызовов** окна и установите или снимите флажок **Показать \<**  _информацией, которая должна_ **>**.

## <a name="bkmk_symbols"></a> Загрузить символы для модуля (C#, C++, Visual Basic, F#)

В окне **Стек вызовов** можно загружать символы отладки для кода, для которого в данный момент нет загруженных отладочных символов. Это могут быть символы платформы .NET Framework или системные символы, загруженные с общедоступных серверов символов корпорации Майкрософт, или символы в каталоге символов на компьютере, на котором производится отладка.

См. статью [Указание файлов символов (.pdb) и файлов с исходным кодом в отладчике Visual Studio](../debugger/specify-symbol-dot-pdb-and-source-files-in-the-visual-studio-debugger.md).

### <a name="to-load-symbols"></a>Чтобы загрузить символы

1.  В **стек вызовов** окно, щелкните правой кнопкой мыши кадр стека для символы не загружены. Кадр затеняется.

2.  Пункты **загрузить символы** , а затем выберите **серверы символов Microsoft** (если доступно), или найдите путь к символам.

### <a name="to-set-the-symbol-path"></a>Установка пути к символам

1.  В окне **Стек вызовов** выберите пункт **Параметры символов** из контекстного меню.

     Появится диалоговое окно **Параметры**, открытое на странице **Символы**.

2.  Выберите **параметры символов**.

3.  В диалоговом окне **Параметры** щелкните значок "Папка".

     В поле **Места размещения файлов символов (.pdb)** появится курсор.

4.  Введите путь к папке расположения символов на компьютере, на котором производится отладка. Для локальной и удаленной отладки, это путь на локальном компьютере.

5.  Выберите **ОК** закрыть **параметры** диалоговое окно.
