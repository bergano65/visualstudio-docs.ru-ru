# <a name="walkthrough-deploy-a-project-task-list-definition"></a>Пошаговое руководство: Развертывание определения списка задач проекта

В этом пошаговом руководстве рассматривается использование [!INCLUDE[vs_dev11_long](../sharepoint/includes/vs-dev11-long-md.md)] для создания, настройки, отладки и развертывания списка SharePoint с целью отслеживания задач проекта.

[!INCLUDE[note_settings_general](../sharepoint/includes/note-settings-general-md.md)]

## <a name="prerequisites"></a>Предварительные требования

- Поддерживаемые редакции Microsoft Windows и SharePoint.

- Visual Studio 2017 или выпуск из Visual Studio Application Lifecycle Management (ALM).

## <a name="create-a-sharepoint-list"></a>Создание списка SharePoint

Создайте проект списка SharePoint и свяжите определение списка с задачами.

1. Откройте **новый проект** диалогового окна разверните узел **SharePoint** узел и нажмите кнопку **2010** узла.

2. В **шаблоны** панели выберите **проект SharePoint 2010** шаблон, имя проекта **ProjectTaskList**, а затем выберите **ОК**кнопки.

     **Мастер настройки SharePoint** отображается.

3. Укажите локального сайта SharePoint, который используется для отладки, выберите **развернуть как решение фермы** переключатель, а затем выберите **Готово** кнопки.

4. Откройте контекстное меню для проекта, а затем выберите **добавить** > **новый элемент**.

5. В **шаблоны** панели выберите **списка** шаблона и нажмите кнопку **добавить** кнопки.

     **Мастер настройки SharePoint** отображается.

6. В **какое имя вы хотите отобразить списка?** введите **списка задач проекта**.

7. Выберите **создать не настраиваемый список, основанный на существующий тип списка** переключатель, а затем в списке, выберите **задачи**, а затем выберите **Готово** кнопки.

     Список, компонентов и пакетов отображаются в **обозревателе решений**.

## <a name="add-an-event-receiver"></a>Добавить приемник событий

В списке задач можно добавить приемник событий, автоматически задающий описание задачи и срок ее выполнения. Следующая процедура добавляет простого обработчика событий в экземпляр списка в качестве приемника событий.

1. Откройте контекстное меню для узла проекта, выберите **добавить**, а затем выберите **новый элемент**.

2. В списке шаблонов SharePoint выберите **приемника событий** шаблона и назовите его **ProjectTaskListEventReceiver**.

     **Мастер настройки SharePoint** отображается.

3. На **Выбор параметров приемника событий** выберите **события элемента списка** тип приемника событий в **тип приемника событий требуются** списка.

4. В **элемент, который должен быть источником события** выберите **задачи**.

5. В списке обрабатываемых событий установите флажок рядом с полем **добавлен элемент**, а затем выберите **Готово** кнопки.

     Новый узел приемника событий добавляется в проект с файлом кода, который называется **ProjectTaskListEventReceiver**.

6. Добавьте код для `ItemAdded` метод в **ProjectTaskListEventReceiver** файл кода. Каждый раз добавляется новая задача, дата оплаты и описание по умолчанию добавляется задача. Значение по умолчанию из-за Дата — 1 июля 2009 г.

## <a name="customize-the-project-task-list-feature"></a>Настроить компонент списка задач проекта

При создании решения SharePoint, Visual Studio автоматически создает компоненты для применяемого по умолчанию элементы проекта. Параметры списка задач проекта для сайта SharePoint можно настроить с помощью конструктора компонентов.

1. В **обозревателе решений**, разверните **функции**.

2. Откройте контекстное меню для **Feature1**, а затем выберите **конструктор представлений**.

3. В **Title** введите **компонент списка задач проекта**.

4. В **область** выберите **Web**.

5. В **свойства** окне введите **1.0.0.0** как значение для **версии** свойство.

## <a name="customize-the-project-task-list-package"></a>Настройка пакета списка задач проекта

При создании проекта SharePoint, Visual Studio автоматически добавляет возможности, которые содержат элементы проекта по умолчанию для пакета. Параметры списка задач проекта для сайта SharePoint можно настроить с помощью конструктора пакетов.

1. В **SolutionExplorer**, откройте контекстное меню для **пакета**, а затем выберите **конструктор представлений**.

2. В **имя** введите **ProjectTaskListPackage**.

3. Выберите **сброс веб-сервера** "флажок".

## <a name="build-and-test-the-project-task-list"></a>Сборка и тестирование списка задач проекта

При запуске проекта открывается сайт SharePoint. Тем не менее нужно вручную перейти в расположение списка задач.

1. Выберите **F5** ключ для создания и развертывания списка задач проекта.

     Откроется сайт SharePoint.

2. Выберите **Главная** вкладки.

3. В левой боковой панели, выберите **списка задач проекта** ссылку.

     Откроется окно списка задач проекта.

4. В **Инструменты списка** вкладке, выберите **элементы** вкладки.

5. В **элементы** группе, выберите **новый элемент** кнопки.

6. В **Title** текста введите **Task1**.

7. Выберите **Сохранить** кнопки.

     После обновления сайта **Task1** появится задача со сроком 7/1/2009.

8. Выберите **Task1**.

     Подробное представление задачи, отобразится описание «Это критически важной задачей».

## <a name="deploy-the-project-task-list"></a>Развертывание списка задач проекта

После создания и тестирования списка задач проекта, можно развернуть его, чтобы *локальной системы* или *удаленной системе*. Локальная система — тот же компьютер, на котором было разработано решение, в то время как удаленная система — это другой компьютер.

### <a name="to-deploy-the-project-task-list-to-the-local-system"></a>Для развертывания списка задач проекта в локальной системе

В строке меню Visual Studio, выберите **построения** > **развернуть решение**.

Visual Studio пула приложений IIS, извлекает все существующие версии решения, копирует пакет решения (*.wsp*) файл в SharePoint и активирует его компоненты. Теперь можно использовать решение в SharePoint. Дополнительные сведения о действиях по настройке развертывания см. в разделе [как: изменение конфигурации развертывания SharePoint](../sharepoint/how-to-edit-a-sharepoint-deployment-configuration.md).

### <a name="to-deploy-the-project-task-list-to-a-remote-system"></a>Для развертывания списка задач проекта в удаленной системе

1. В строке меню Visual Studio, выберите **построения** > **публикации**.

2. В **публикации** диалоговом окне выберите **опубликовать в файловой системе** переключатель.

     Можно изменить целевое расположение в **публикации** диалоговое окно, выбрав кнопку с многоточием ![значок многоточия](../sharepoint/media/ellipsisicon.gif "значок многоточия") и перейдя в другое расположение.

3. Выберите **публикации** кнопки.

     Объект *.wsp* файл создается для решения.

4. Копировать *.wsp* файла к удаленной системе SharePoint.

5. С помощью PowerShell `Add-SPUserSolution` команду, чтобы установить пакет на удаленную установку SharePoint. (Для решения фермы используйте `Add-SPSolution` команды.)

     Например, `Add-SPUserSolution C:\MyProjects\ProjectTaskList\ProjectTaskList\bin\Debug\ProjectTaskList.wsp`.

6. С помощью PowerShell `Install-SPUserSolution` команду, чтобы развернуть решение. (Для решения фермы используйте `Install-SPSolution` команды.)

     Например, `Install-SPUserSolution -Identity ProjectTaskList.wsp -Site http://NewSiteName`.

     Дополнительные сведения об удаленном развертывании см. в разделе [решений с помощью](http://go.microsoft.com/fwlink/?LinkId=217680) и [Добавление и развертывание решений в SharePoint 2010 с помощью PowerShell](http://go.microsoft.com/fwlink/?LinkId=217682).
