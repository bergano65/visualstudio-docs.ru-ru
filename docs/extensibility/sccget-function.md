# <a name="sccget-function"></a>Функция SccGet
Эта функция извлекает копию одного или нескольких файлов для просмотра и компиляции, но не для изменения. В большинстве систем файлы помечаются как доступные только для чтения.

## <a name="syntax"></a>Синтаксис

```cpp
SCCRTN SccGet(
   LPVOID    pvContext,
   HWND      hWnd,
   LONG      nFiles,
   LPCSTR*   lpFileNames,
   LONG      fOptions,
   LPCMDOPTS pvOptions
);
```

### <a name="parameters"></a>Параметры
 pvContext

[in] Структура контекста подключаемый модуль системы управления версиями.

 hWnd

[in] Дескриптор окна интегрированной среды разработки, подключаемый модуль системы управления версиями можно использовать в качестве родительского для любой диалоговых окон, которые он предоставляет.

 nFiles

[in] Число файлов, указанных в `lpFileNames` массива.

 lpFileNames

[in] Массив полных имен файлов, которые требуется получить.

 Возможности

[in] Команда флаги (`SCC_GET_ALL`, `SCC_GET_RECURSIVE`).

 pvOptions

[in] Параметры конкретного подключаемого модуля системы управления версиями.

## <a name="return-value"></a>Возвращаемое значение
 Подключаемый модуль реализации элемента управления источника этой функции должен возвращать одно из следующих значений:

|Значение|Описание|
|-----------|-----------------|
|SCC_OK|Успешное выполнение операции get.|
|SCC_E_FILENOTCONTROLLED|Файл не существует в системе управления версиями.|
|SCC_E_OPNOTSUPPORTED|Система управления версиями не поддерживает эту операцию.|
|SCC_E_FILEISCHECKEDOUT|Не удается получить файл, который пользователь в данный момент извлечен.|
|SCC_E_ACCESSFAILURE|Возникла проблема с доступом к системе управления версиями, возможно, из-за проблемы с сетью или конфликтов. Рекомендуется повторить операцию.|
|SCC_E_NOSPECIFIEDVERSION|Задана недопустимая версия или даты и времени.|
|SCC_E_NONSPECIFICERROR|Обнаружена неспецифическая ошибка; файл не был синхронизирован.|
|SCC_I_OPERATIONCANCELED|Операция отменена до завершения.|
|SCC_E_NOTAUTHORIZED|Пользователь не авторизован для выполнения этой операции.|

## <a name="remarks"></a>Примечания
 Эта функция вызывается с числом и массив имен файлов, которые требуется получить. Если интегрированная среда разработки передает флаг `SCC_GET_ALL`, это означает, что элементы в `lpFileNames` не файлы, но каталоги, и все файлы в системе управления версиями в заданной каталоги должны быть получены.

 `SCC_GET_ALL` Флаг может сочетаться с `SCC_GET_RECURSIVE` для получения всех файлов в заданной каталогов и все подкаталоги.

> [!NOTE]
>  `SCC_GET_RECURSIVE` никогда не должен передаваться без `SCC_GET_ALL`. Кроме того, обратите внимание, что если каталоги *C:\A* и *C:\A\B* передаются в инструкцию get рекурсивного *C:\A\B* и всех его подкаталогах фактически извлекается дважды. Ответственность за IDE — и не система управления подключаемого модуля, чтобы убедиться в том, что повторяющиеся значения, например, хранятся из массива.

 Наконец, даже если системы управления версиями подключаемого модуля указан `SCC_CAP_GET_NOUI` флаг при инициализации, что он не имеет пользовательского интерфейса для команды Get, эта функция по-прежнему могут быть вызваны интегрированной среды разработки для получения файлов. Флаг просто означает, что интегрированной среды разработки не отображается пункт меню Get, и что подключаемый модуль не должен предоставлять никаких элементов пользовательского интерфейса.

## <a name="rename-files-and-sccget"></a>Переименование файлов и SccGet
 Ситуации: пользователь извлекает файл, например, *a.txt*и обновляет его. Прежде чем *a.txt* могут быть возвращены, второй пользователь его переименовывает *a.txt* для *b.txt* в базе данных управления версиями, извлекает *b.txt*, делает Некоторые изменения в файл и возврате файла. Первый пользователь хочет изменения, внесенные пользователем второго, первый пользователь его переименовывает их локальную версию *a.txt* файл *b.txt* и выполняет инструкцию get для файла. Тем не менее локальный кэш, который отслеживает номера версий по-прежнему считает первой версии *a.txt* хранится локально и системы управления версиями не удается разрешить различия.

 Существует два способа для решения этой ситуации, когда становится синхронизированы с источником данных элемента управления в локальном кэше версией исходного элемента управления:

1. Не разрешать переименование файла в базе данных управления версиями, который в данный момент извлечен.

2. Сделайте эквивалентом «удалить старые» следуют «добавить». Следующий алгоритм — один из способов выполнения этой задачи.

    1. Вызовите [SccQueryChanges](../extensibility/sccquerychanges-function.md) функции, чтобы узнать о переименование *a.txt* для *b.txt* в базе данных управления версиями.

    2. Переименование локальной *a.txt* для *b.txt*.

    3. Вызовите `SccGet` функции для обоих *a.txt* и *b.txt*.

    4. Так как *a.txt* не существует в базе данных управления версиями, локальная версия кэша удаляется отсутствующих *a.txt* сведения о версии.

    5. *B.txt* извлеченного файла объединяется с содержимым локального *b.txt* файла.

    6. Обновленный *b.txt* вернуть файл.
