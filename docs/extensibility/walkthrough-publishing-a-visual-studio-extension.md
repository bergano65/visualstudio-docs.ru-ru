# <a name="walkthrough-publish-a-visual-studio-extension"></a>Пошаговое руководство: Публикация расширения Visual Studio

В этом пошаговом руководстве показано, как для публикации расширения Visual Studio в Visual Studio Marketplace. При добавлении расширения в Marketplace, разработчики могут использовать **расширения и обновления** поиск новых и обновленных расширений.

## <a name="prerequisites"></a>Предварительные требования

 Для выполнения этого пошагового руководства необходимо установить пакет SDK для Visual Studio. Дополнительные сведения см. в разделе [установить пакет SDK для Visual Studio](../extensibility/installing-the-visual-studio-sdk.md).

## <a name="create-a-visual-studio-extension"></a>Создание расширения Visual Studio

В этой статье используется расширение VSPackage по умолчанию, но действия являются допустимыми для каждого типа расширений.

1. Создание пакета VSPackage в C# с именем `TestPublish` , с командой меню. Дополнительные сведения см. в разделе [создать свое первое расширение: Hello World](../extensibility/extensibility-hello-world.md).

## <a name="package-your-extension"></a>Пакет расширения

1. Обновить расширение *.vsixmanifest* с правильными данными о название продукта, автор и версии.

   ![Обновить расширение vsixmanifest](media/update-extension-vsixmanifest.png)

2. Построение расширения **выпуска** режим. Теперь расширение представлена в виде VSIX в папке \bin\Release.

3. Дважды щелкните VSIX для проверки установки.

## <a name="test-the-extension"></a>Тестирование расширения

 Перед распространением расширения, сборки и тестирования, чтобы убедиться, что он правильно установлен в экспериментальном экземпляре Visual Studio.

1. В Visual Studio начните отладку, чтобы открыть экспериментальный экземпляр Visual Studio.

2. В экспериментальном экземпляре выберите **средства** меню и выберите пункт **расширения и обновления**. Расширение TestPublish в центральной области появится и включить.

3. На **средства** меню, убедитесь, что вы видите команду теста.

## <a name="publish-the-extension-to-the-visual-studio-marketplace"></a>Публикация расширения для Visual Studio Marketplace

1. Убедитесь, что вы создавали версии расширения, и оно не устаревает.

2. Откройте в веб-браузер, [Visual Studio Marketplace](https://marketplace.visualstudio.com/vs) веб-сайта.

3. В правом верхнем углу щелкните **вход**.

4. Войдите с помощью своей учетной записи Майкрософт. Если у вас нет учетной записи Майкрософт, его можно создать на этом этапе.

5. Нажмите кнопку **публикации расширения**.  Этот параметр, можно перейти к странице Управление для всех расширений. Если у вас нет учетной записи издателя, вам будет предложено создать его в данный момент.

   ![Отправка в Marketplace](media/upload-to-marketplace.png)

6. Выберите издателя, которого вы хотите использовать для отправки расширения. Издатели можно изменить, щелкнув имена издателей, перечисленные в левой части. Щелкните **новое расширение** и выберите **Visual Studio**.

7. В **1: отправить расширение**, вы можете отправить VSIX-файл непосредственно в Visual Studio Marketplace или просто добавить ссылку на свой веб-сайт. В этом примере расширение, *TestPublish.vsix* загружается. Перетащите модуль или использовать **щелкните** ссылку, чтобы перейти к файлу. Найти свое расширение в папке \bin\Release проекта.  Нажмите кнопку **Продолжить**.

8. В **2: укажите сведения о расширении**, некоторые поля заполняются автоматически из *source.extension.vsixmanifest* файла из расширения. Найти дополнительные сведения о каждом ниже:

    * **Внутреннее имя** используется в URL-адрес страницы сведений о расширении. Например, публикации расширения в разделе имя издателя «myname» с указанием внутреннего имени как «Мои расширения» приводит к URL-адрес «marketplace.visualstudio\.com/items?itemName=myname.myextension» подробности свое расширение страница.
    
    * **Отображаемое имя** вашего расширения. Это имя автоматически подставляется из *source.extension.vsixmanifest* файл.
   
    * **Версия** чисел при отправке расширения. Этой версии автоматически подставляется из *source.extension.vsixmanifest* файл.
    
    * **Идентификатор VSIX** — уникальный идентификатор, который используется для расширения Visual Studio. Этот идентификатор является обязательным, если вы хотите иметь расширение обновлены автоматически. Этот идентификатор является автоматически подставляется из *source.extension.vsixmanifest* файл.
    
   * **Логотип** , используемый для расширения. Этот логотип автоматически подставляется из *source.extension.vsixmanifest* файла, если указано.
    
     * **Краткое описание** того, что делает ваше расширение. Это описание будет автоматически подставляется из *source.extension.vsixmanifest* файл.
    
     * **Общие сведения о** хорошо подходит для включения снимки экрана и подробные сведения о модуле.
    
     * **Поддерживаемые версии Visual Studio** позволяет выбрать, какие версии Visual Studio расширение будет работать на. Расширение устанавливается только для этих версий.
    
     * ** Поддерживается edition позволяет выбрать, какие выпуски Visual Studio, расширение будет работать в Visual Studio. Расширение устанавливается только для этих выпусков.
    
     * **Тип**. Наиболее распространенный тип расширений являются **средства**.
    
     * **Категории**. Выберите до трех файлов, которые лучше всего подходит для вашего расширения.
    
     * **Теги** являются ключевые слова, которые помогают пользователям найти ваше расширение. Теги можно повысить Релевантность поиска расширений в Marketplace.
    
     * **Ценовая категория** затраты на расширение.
    
     * **Репозиторий исходного кода** позволяет предоставить ссылку на исходный код с сообществом.
    
     * **Разрешить вопросы И ответы для модуля** позволяет оставить вопросы на странице входа расширения.

9. Нажмите кнопку **сохранить и отправить**. Страница управления этот параметр принимает, обратно на издатель. Расширение еще не опубликован. Чтобы опубликовать расширение, щелкните правой кнопкой мыши для расширения и выбрать **сделать общедоступным**. Можно просмотреть, как расширения будет выглядеть в Marketplace, выбрав **Просмотр расширения**. Для получения числа, щелкните **отчеты**. Чтобы изменить расширение, щелкните **изменить**.

   ![Запись меню расширения](media/extension-entry-menu.png)

10. После нажатия кнопки **сделать общедоступным**, расширение теперь является общедоступным. Поиск в Visual Studio Marketplace, для модуля.

## <a name="add-additional-users-to-manage-your-publisher-account"></a>Добавить дополнительных пользователей для управления своей учетной записи издателя

Marketplace поддерживает предоставление дополнительным пользователям разрешений для доступа и управления учетную запись издателя.

1. Перейдите к учетной записи издателя, который вы хотите добавить дополнительных пользователей.

2. Выберите **члены** и щелкнуть **добавить**.

   ![Добавить дополнительных пользователей](media/add-users.png)

3. Затем можно указать адрес электронной почты пользователя, который вы хотите добавить и предоставить необходимый уровень доступа в рамках **выберите роль**.  Можно выбрать один из следующих вариантов.

   * **Создатель**: пользователь публикации расширения, но не может просматривать и управлять расширения, опубликованные другими пользователями.
  
   * **Модуль чтения**: пользователь может просматривать расширения, но нельзя опубликовать или Управление расширениями.
  
   * **Участник**: пользователь может публиковать и Управление расширениями, но не может изменять параметры издателя или Управление доступом.
  
   * **Владелец**: пользователь может опубликовать и Управление расширениями, изменять параметры издателя и управление доступом.
  
## <a name="install-the-extension-from-the-visual-studio-marketplace"></a>Установите расширение из Visual Studio Marketplace

Теперь, когда публикуется расширения, установите его в Visual Studio и протестируйте его.

1. В Visual Studio на **средства** меню, щелкните **расширения и обновления**.

2. Нажмите кнопку **Online** и выполните поиск **TestPublish**.

3. Нажмите **Загрузить**. Затем расширение запланировано к установке.

4. Чтобы завершить установку, закройте все экземпляры Visual Studio.

## <a name="remove-the-extension"></a>Удаление расширения

Расширение можно удалить из Visual Studio Marketplace и на компьютере.

### <a name="to-remove-the-extension-from-the-visual-studio-marketplace"></a>Чтобы удалить расширение из Visual Studio Marketplace

1. Откройте [Visual Studio Marketplace](https://marketplace.visualstudio.com/vs) веб-сайта.

2. В правом верхнем углу щелкните **публикации** расширения. Выберите издателя, который использовался для публикации **TestPublish**. В описании **TestPublish** отображается.

3. Щелкните правой кнопкой мыши на запись в списке и нажмите кнопку **удалить**. Вам будет предложено подтвердить удаление расширения. Нажмите кнопку **ОК**.

### <a name="to-remove-the-extension-from-your-computer"></a>Чтобы удалить расширение с компьютера

1. В Visual Studio на **средства** меню, щелкните **расширения и обновления**.

2. Выберите **TestPublish** и нажмите кнопку **удаления**. Затем расширение запланировано к удалению.

3. Чтобы завершить удаление, закройте все экземпляры Visual Studio.
